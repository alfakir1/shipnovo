# دليل مستخدمي نظام ShipNovo وواجهاتهم

يوضح هذا المستند أنواع المستخدمين في نظام ShipNovo والأدوار المناطة بكل منهم، بالإضافة إلى شرح للواجهات البرمجية (Dashboards) الخاصة بكل فئة.

## 1. أنواع المستخدمين (User Roles)

ينقسم مستخدمو النظام إلى أربع فئات رئيسية كما هو محدد في قاعدة البيانات:

| الدور (Role) | المسمى الوظيفي | الوصف |
| :--- | :--- | :--- |
| **Admin** | المدير العام | يمتلك صلاحيات كاملة لإدارة النظام، الإعدادات، والمستخدمين. |
| **Ops** | مسؤول العمليات | يمثل محرك المنصة (Orchestration Engine)، حيث يقوم بإدارة الشحنات وربطها بالشركاء وتحديث الحالات. |
| **Customer** | العميل (التاجر/المرسل) | هو صاحب البضاعة الذي يقوم بطلب خدمات الشحن ومتابعتها. |
| **Partner** | الشريك اللوجستي (ناقل/مخلص) | الجهات الخارجية التي تنفذ المهام (مثل شركات النقل أو مكاتب التخليص الجمركي). |

---

## 2. شرح الواجهات حسب المستخدم

### أ. واجهة العميل (Customer Dashboard)
*   **المسار:** `/customer/dashboard`
*   **الوظائف الرئيسية:**
    *   **طلب شحنة جديدة:** نموذج متعدد الخطوات لإدخال تفاصيل الشحنة (المنشأ، الوجهة، نوع البضاعة).
    *   **متابعة الشحنات:** واجهة عرض تفصيلية لكل شحنة مع جدول زمني (Timeline) للحالات.
    *   **إدارة المستندات:** رفع وتحميل الفواتير والمستندات الجمركية.
    *   **مركز الدعم:** فتح وإدارة التذاكر (Tickets) للتواصل مع فريق العمليات.

### ب. واجهة العمليات (Ops Dashboard)
*   **المسار:** `/ops/dashboard`
*   **الوظائف الرئيسية:**
    *   **لوحة التحكم العالمية:** نظرة عامة على كافة الشحنات النشطة في النظام.
    *   **إدارة عرض الأسعار (Quotation):** مراجعة عروض الأسعار المقترحة أو تعديلها.
    *   **تعيين الشركاء:** اختيار الشريك المناسب (Partner) لكل خطوة من خطوات الشحنة.
    *   **التدخل اليدوي:** تعديل حالات الشحنة في حال وجود تأخير أو مشاكل تقنية.

### ج. واجهة الشريك (Partner Dashboard)
*   **المسار:** `/partner/dashboard`
    *   **الوظائف الرئيسية:**
        *   **قائمة المهام:** عرض الشحنات التي تم تعيينها لهذا الشريك فقط.
        *   **تحديث الحالة:** واجهة سريعة لتأكيد استلام البضاعة أو توصيلها.
        *   **إثبات التسليم (POD):** رفع صور أو مستندات تثبت نجاح عملية التوصيل أو التخليص.

### د. واجهة الإدارة (Admin Dashboard)
*   **المسار:** `/admin`
*   **الوظائف الرئيسية:**
    *   **إدارة المستخدمين:** تفعيل/تعطيل حسابات الموظفين والعملاء.
    *   **إعدادات النظام:** التحكم في التسعير، المناطق الجغرافية، والاشتراكات.
    *   **التقارير المالية:** متابعة الفواتير والتحصيلات على مستوى المنصة ككل.

---

## 3. تدفق العمل (Workflow Example)
1. يقوم **العميل** بطلب شحنة.
2. تظهر الشحنة في واجهة **مسؤول العمليات** للمراجعة وإسناد المهام.
3. يستلم **الشريك** إشعاراً في واجهته الخاصة للبدء في التنفيذ.
4. يراقب **المدير (Admin)** الأداء العام للتأكد من سلاسة العمل.
